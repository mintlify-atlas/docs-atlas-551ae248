The RPC API enables you to query status information for nodes and validators.

## Quick Reference

| Method | Parameters | Description |
|--------|------------|-------------|
| [`status`](#node-status) | _none_ | Returns general status of a given node and validator set |
| [`network_info`](#network-info) | _none_ | Returns current state of node network connections |
| [`validators`](#validation-status) | `epoch_id` OR `[null]` | Returns active validators on the network |

## Node Status

Returns general status of a given node (sync status, nearcore node version, protocol version, etc.), and the current set of validators.

<ParamField query="method" type="string" required>
  `status`
</ParamField>

<ParamField query="params" type="array" required>
  `[]`
</ParamField>

<CodeGroup>
```json JSON
{
  "jsonrpc": "2.0",
  "id": "dontcare",
  "method": "status",
  "params": []
}
```

```javascript JavaScript
import { JsonRpcProvider } from "near-api-js";

const provider = new JsonRpcProvider({
    url: "https://test.rpc.fastnear.com",
});

const response = await provider.status();
```

```bash HTTPie
http POST https://rpc.testnet.near.org \
  jsonrpc=2.0 \
  id=dontcare \
  method=status \
  params:='[]'
```
</CodeGroup>

## Validation Status

Queries active validators on the network returning details and the state of validation on the blockchain.

<ParamField query="method" type="string" required>
  `validators`
</ParamField>

<ParamField query="params" type="array | object" required>
  `[null]` for current validators, or `{"epoch_id": "..."}` for specific epoch
</ParamField>

<Note>
You can obtain the `epoch_id` from a block that belongs to a specific epoch. If you want to retrieve the current list of validators, pass `null` as the parameter. Additionally, you can query validators for past epochs by providing the `epoch_id` of the desired past epoch.
</Note>

### Query current validators

<CodeGroup>
```json JSON
{
  "jsonrpc": "2.0",
  "id": "dontcare",
  "method": "validators",
  "params": [null]
}
```

```bash HTTPie
http POST https://rpc.testnet.near.org \
  jsonrpc=2.0 \
  id=dontcare \
  method=validators \
  params:='[null]'
```
</CodeGroup>

### Query by epoch_id

<CodeGroup>
```json JSON
{
  "jsonrpc": "2.0",
  "id": "dontcare",
  "method": "validators",
  "params": {
    "epoch_id": "94jeudySZcxGBSVgKXn3xPT3P5iFF6YcnxC43F15QtkQ"
  }
}
```

```bash HTTPie
http POST https://rpc.testnet.near.org \
  jsonrpc=2.0 \
  id=dontcare \
  method=validators \
  params:='{"epoch_id": "94jeudySZcxGBSVgKXn3xPT3P5iFF6YcnxC43F15QtkQ"}'
```
</CodeGroup>

## Error Handling

### Common Error Types

| Error Code | Description | Solution |
|------------|-------------|----------|
| `UnknownEpoch` | Epoch ID not found | Use a valid epoch ID from existing blocks |
| `InvalidAccount` | Invalid account format | Use valid account ID format (e.g., `account.near`) |
| `RequestTimeout` | Request timed out | Retry the request or use a different RPC endpoint |
| `InternalError` | Server-side error | Retry the request after a short delay |
| `MethodNotFound` | Invalid method name | Check method spelling and API version |

## Best Practices

- **Cache validator data**: Validator information changes infrequently, consider caching for several minutes
- **Use specific queries**: Request only the data you need to minimize response size
- **Monitor network health**: Use `status` endpoint to check node sync status before other operations
